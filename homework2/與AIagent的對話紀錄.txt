與 AI 助手的開發對話紀錄（詳細版）

目標
----
將 HW1-2（GridWorld: 策略與價值顯示）實作為一個互動網頁，並保留 HW1-1（homework1_site）不變。網頁需支援：
- 在格子上設定 Start / End / Obstacles
- 產生 Random Policy 與 Optimal Policy
- 執行 Policy Evaluation 顯示 Value matrix
- 提供 Debug Overlay 協助診斷 -10.00 之類的數值飽和問題

關鍵討論與決策
----------------
1) 發現問題（根因分析）
   - 現象：多數格子評估為 -10.00
   - 原因：當策略導致自我迴圈（stay in place）且 step reward = -1, gamma = 0.9，則 V = R/(1-gamma) = -10 為吸引子。

2) 解法（分步實作）
   - 加入 reachability：從終點向外 BFS 標記可達格子，UI 顯示 unreachable 為 '-'
   - 避免自我迴圈：在隨機策略與 greedy 抽取時優先選擇非 self-loop 動作；若無可選動作則將 policy[s] 設為 null
   - distance 啟發：用 BFS 計算每格到終點的最短步數，並以 distance 作為 policy evaluation / value iteration 的初始 guess（可加速收斂）
   - Gauss–Seidel：將迭代改為 in-place 更新，增加穩定性與收斂速度
   - Debug Overlay：顯示每格的 distance、policy、reachable 與 self-loop 標記，並可在 console 輸出關鍵陣列（便於複製貼上給協助者）

3) UX / 介面決定
   - Grid size 限制 5–9（避免過大造成畫面或演算法不易觀察）
   - Obstacle 數量限制為 n-2，並在 UI 顯示剩餘可放置的障礙數
   - 提供 static preview (`index_static.html`) 以便在無法啟動 Flask 時仍可測試前端
